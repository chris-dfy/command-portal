# Design System

This presentational system follows the [NEXUS Platform Constitution](../architecture/NEXUS_Platform_Constitution.md).

The available generated portal and current desktop frontend both use a restrained operational interface: deep navy surfaces, fine cyan borders, compact uppercase kickers, monospace evidence IDs, cyan intelligence cues, amber limitations, green verified states, and red unavailable states.

## Tokens

- Background: `#050b10`
- Panel: translucent `#09141d`
- Primary text: `#eaf5fb`
- Muted text: `#8ba4b4`
- Accent: `#62d2ff`
- Verified: `#6ee7b7`
- Limited/pending: `#f2c66d`
- Failed/unavailable: `#fb9292`
- Corners: 7–13px; pills are fully rounded
- Borders: one-pixel, low-contrast cyan; never glow without a semantic reason

Typography uses the system sans stack; IDs and machine records use the system monospace stack. Headings are light-to-medium weight. Dense operational labels use uppercase with restrained tracking.

Color never carries status alone: every status includes text and a visible dot or icon.
