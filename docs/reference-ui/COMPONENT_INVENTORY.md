# Component Inventory

| Component | Purpose | Data behavior |
|---|---|---|
| Status rail | Global source, mode, connection, truth posture | Snapshot metadata only |
| Fixture banner | Makes non-live data unmistakable | Required in fixture mode |
| Area navigation | Six required portal areas | Client-only selection |
| Status pill | Text, dot, and semantic tone | Never upgrades source state |
| Data panel | Bounded section with kicker and title | Renders a curated domain view |
| Metric grid | High-signal operational facts | Displays `unavailable` when absent |
| Definition list | Key/value runtime posture | Does not infer missing values |
| Verification matrix | Capability result and state | Preserves staged/limited states |
| Evidence list | Claims, proofs, receipts | GET detail drill-down only |
| Specialist registry | Maturity and trained-model distinction | Keeps `actualTrainedSLMs=0` |
| Asset coverage | Manifest, dependency, and gap disclosure | Includes limitations/missing requirements |
| Error/empty state | Honest unavailable or missing data | Never substitutes fixture data |
| Evidence dialog | Validated detail record | Same-origin allowlisted GET |

Shared components are product-name-neutral. Visible identity is supplied by brand configuration.
